# Лекция №2. Работа с удалённым репозиторием. Ветвление

![Вомбат](https://cithub.ru/api/files/вомбат.jpg)

---

## 1. Удалённый репозиторий

> **Удалённый репозиторий** — копия проекта, хранящаяся на сервере (GitHub, GitLab) и доступная по сети.

- Полноценный репозиторий со своей историей и ветками  
- Позволяет совместную работу, синхронизацию и резервное копирование

---

## 2. Подключение удалённого репозитория

> **Команда `git remote add`** — добавляет ссылку на удалённый репозиторий.

**Формат:**  
```bash
git remote add <имя> <URL>
```

**Пример:**  
```bash
git remote add origin https://github.com/CIT2025/4335Group-26
```

![Получение URL-адреса репозитория](https://cithub.ru/api/files/вомбат.jpg)

---

## 3. Отключение удалённого репозитория

> **Команда `git remote remove`** — удаляет связь с удалённым репозиторием.

**Формат:**  
```bash
git remote remove <имя>
```

**Пример:**  
```bash
git remote remove origin
```

> Не удаляет сам репозиторий на сервере.

---

## 4.1. Переименование удалённого репозитория

> **Команда `git remote rename`** — изменяет локальное имя удалённого репозитория.

**Формат:**  
```bash
git remote rename <старое> <новое>
```

**Пример:**  
```bash
git remote rename origin upstream
```

---

## 4.2. Просмотр удалённых репозиториев

> **Команда `git remote show`** — отображает список или информацию об удалённых репозиториях.

**Примеры:**  
```bash
git remote show
git remote show origin
```

---

## 5. Клонирование удалённого репозитория

> **Команда `git clone`** — создаёт локальную копию удалённого репозитория.

**Формат:**  
```bash
git clone <URL>
```

**Пример:**  
```bash
git clone https://github.com/CIT2025/4335Group-26
```

**Что происходит:**  
- Создаётся папка проекта  
- Копируется вся история  
- Добавляется `origin`  
- Переключается на основную ветку

---

## 6. Получение изменений: `git fetch`

> **Команда `git fetch`** — загружает изменения из удалённого репозитория, но не обновляет рабочий каталог.

**Формат:**  
```bash
git fetch [имя]
```

**Пример:**  
```bash
git fetch origin
```

> Обновляет только удалённые ветки (например, `origin/develop`).

---

## 6.2. Применение изменений после `fetch`

После `git fetch`:

```bash
git checkout develop
git merge origin/develop
```

Или используйте `git pull`.

---

## 7. Получение и обновление: `git pull`

> **Команда `git pull`** — выполняет `fetch` + `merge`.

**Формат:**  
```bash
git pull [репозиторий] [ветка]
```

**Пример:**  
```bash
git pull origin main
```

> Стандартный способ синхронизации локальной ветки с удалённой.

---

## 8. Отправка изменений: `git push`

> **Команда `git push`** — загружает локальные коммиты в удалённый репозиторий.

**Формат:**  
```bash
git push [репозиторий] [ветка]
```

**Пример:**  
```bash
git push origin develop
```

**Рекомендации:**  
- Всегда делайте `pull` перед `push`  
- Используйте `--force-with-lease` вместо `--force`

---

## 9.1. Определение ветки

> **Ветка в Git** — указатель на конкретный коммит.

- Логически — независимая линия разработки  
- Физически — файл в `.git/refs/heads/`

---

## 9.2. Назначение веток

- Параллельная разработка  
- Эксперименты без риска для основного кода  
- Поддержка нескольких версий продукта

---

## 9.3. Создание и переключение

**Создать и переключиться:**  
```bash
git checkout -b feature
```

**Или по отдельности:**  
```bash
git branch feature
git checkout feature
```

---

## 9.4. Просмотр веток

```bash
git branch      # локальные
git branch -r   # удалённые
git branch -a   # все
```

---

## 9.5. Переключение между ветками

```bash
git checkout develop
git checkout -      # вернуться назад
```

> Убедитесь, что нет несохранённых изменений (`git status`).

---

## 9.6. Просмотр истории

```bash
git log --all --graph --oneline
git log develop..feature
```

---

## 10. Сравнение изменений: `git diff`

> **Команда `git diff`** — показывает различия между файлами, коммитами или ветками.

**Примеры:**  
```bash
git diff                    # рабочий каталог vs индекс
git diff --staged           # индекс vs последний коммит
git diff main..feature      # сравнить ветки
git diff коммит1 коммит2    # сравнить коммиты
```

---

## 11. Удаление веток

> **`git branch -d`** — безопасное удаление (только если слита).  
> **`git branch -D`** — принудительное удаление.

**Пример:**  
```bash
git checkout main
git branch -d feature
```

> Нельзя удалить текущую ветку.
