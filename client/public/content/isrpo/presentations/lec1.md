# Лекция №1. Обзор и классификация современных инструментальных средств разработки ПО

![Вомбат](https://cithub.ru/api/files/вомбат.jpg)

---

## 1. Понятие и состав инструментальных средств

> **Инструментальные средства разработки ПО (ИСРПО)** — программное обеспечение, предназначенное для автоматизации всех этапов жизненного цикла программного продукта: от написания кода до тестирования, внедрения и сопровождения.

Инструментальные средства классифицируются по форме реализации:

- **Software tools** — набор автономных программ, каждая из которых решает узкую задачу  
- **IDE** — интегрированная среда разработки с графическим интерфейсом  
- **SDK** — комплект средств разработки для конкретной платформы или оборудования

---

## 2.1. Редакторы исходного текста

Программы для ввода и правки кода.

- Поддержка IntelliSense  
- Подсветка синтаксиса  
- Интеграция с системами контроля версий

*Примеры:* Visual Studio Code, Sublime Text, JetBrains Fleet.

---

## 2.2. Трансляторы: компиляторы и интерпретаторы

Служат для перевода исходного кода в машинный.

- **Компиляторы** — переводят весь код целиком до запуска  
  *Языки:* C++, Rust, Go  
- **Интерпретаторы** — выполняют код построчно во время запуска  
  *Языки:* Python, JavaScript, PHP

---

## 2.3. Программы-компоновщики (линковщики)

Осуществляют сборку программы из отдельных объектных модулей и библиотек.

- Связывают вызовы функций с их реализацией  
- Формируют целостный исполняемый файл

---

## 2.4. Программы-отладчики (дебаггеры)

Инструменты для поиска и исправления логических ошибок.

- Пошаговое выполнение  
- Точки остановки (breakpoints)  
- Отслеживание состояния памяти в реальном времени

---

## 2.5. Средства создания инсталляторов и контейнеризация

Программы для подготовки продукта к передаче пользователю.

- **Классические инсталляторы:** Inno Setup, NSIS  
- **Современный подход:** Docker-контейнеры — упаковка программы со всем окружением

---

## 2.6. Системы документации и веб-сервисы

Средства создания справочных систем и технического описания.

- **Локальные:** Dr.Explain, Doxygen  
- **Веб-ориентированные:** GitBook, GitHub Pages, Read the Docs

---

## 2.7. Средства анализа и реверс-инжиниринга

Инструменты для исследования структуры готового ПО.

- Дизассемблеры (IDA Pro)  
- Декомпиляторы  
- Редакторы ресурсов

---

## 3. CASE-технологии (Computer-Aided System Engineering)

> **CASE-средства** — высокоуровневые программные комплексы, поддерживающие коллективную разработку сложных систем.

**Основные категории:**  
- Визуальное проектирование (UML, PlantUML)  
- Проектирование баз данных (ERwin, MySQL Workbench)  
- Системы контроля версий (Git, GitHub, GitLab)  
- Управление проектами (Jira, YouTrack)

---

## 4. Современные технологические тренды

1. **AI-Driven Development**  

ИИ-ассистенты (GitHub Copilot) дописывают код и генерируют тесты.

2. **Cloud-Native и Remote Development**  

Разработка в облаке через браузер (GitHub Codespaces).

3. **Low-code / No-code платформы**  

Сборка приложений из готовых блоков.

4. **DevOps и CI/CD**  

Автоматизация сборки, тестирования и развёртывания при каждом коммите.
